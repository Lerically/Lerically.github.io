Execute System

Enemy Perspective: Each time the player's next attack kills an enemy, the enemy will become executable, the blood bar will become invisible, and you will get a random number from 1 to 4 which determines your execution animation. The enemy and the player will not be able to move for a while and will not be able to input any actions. This function gives the player the enemy's position and a matching decision animation. Player Perspective: After receiving information about the enemy's perspective, the player will pop up a QTE event before killing the enemy and perform a bullet time. After pressing a certain key during the bullet time, it will start to check if the player's status is suitable for killing, if it is true, the player will not be able to input any valid inputs. And through the enemy's position to adjust the player's decapitation position, through the motion warp to do the player's displacement, and adjust the player's camera, after the end of the animation reset all the functions and status.

Lock-on System

The lock-on system is triggered by pressing a designated button, initiating a line trace from the player's camera to detect enemies in front of them. If an enemy is detected, the camera locks onto the target, aligning the player's focus with the enemy. Simultaneously, the system evaluates all enemies within a specific radius using a cosine calculation to determine their relative position. The player can then swing the mouse left or right to seamlessly switch the lock between nearby enemies, ensuring smooth and intuitive target transitions.

System Overview

The mission system is designed using data tables, structures, and enumerations. Each NPC has a unique ID and name. By listing the NPCs responsible for submitting and receiving each mission in the data table, the missions are logically linked. The mission logic is implemented within the Widget, and specific missions are assigned during gameplay. Mission types include Collect, Annihilate, and Arrive. Each mission provides completion rewards and includes a visual mission guide for players.

Weapon Wheel System

Weapon Wheel System

The weapon wheel allows players to switch weapons interactively via a widget using textures. A radial gradient exponential creates a base circle, with layers superimposed for additional effects. A custom rotator and vector-to-radial conversion enable selection of specific circle sections. Highlighting is achieved by combining these operations with another material that isolates ring segments. This dynamic system provides a seamless and visually appealing weapon selection experience.

Overview

The attack system uses structures, enumerations, and data tables to manage specific attack types—light, heavy, and dash. Different weapons have unique light and heavy attacks, and combinations of these inputs can lead to distinct combo finishers. Light attacks are mapped to the left mouse button and heavy attacks to the right. A custom function records each player input, determining whether it's a light or heavy attack, and matches it to the corresponding entry in a data table. This table includes a map variable that defines potential attack sequences based on weapon type, such as "longsword light" or "stick light." When a longsword attack is initiated, the system recognizes the input, plays the corresponding animation montage, and sets a timing window during which the next attack input is allowed. If another input is made within this window, the system checks the data table for valid combos (e.g., light or heavy follow-ups) and seamlessly transitions to the next attack, maintaining the fluidity of the combo system.

Attack Feedback System

Overview

The attack feedback system enhances combat immersion with frame delay, camera shake, VFX, and enemy animation responses. Frame jerking is implemented using montage play rate adjustments, creating a 0.1-second pause upon each successful hit to amplify the sense of impact. Player camera shake is handled using the legacy camera shake system, adding dynamic movement to emphasize strikes. VFX are precisely spawned at the weapon's sphere trace hit point, visually aligning with the attack's target. Enemy hit animations are given priority over all other montages, ensuring responsive and clear reactions that reinforce the intensity and feedback of the combat experience.
